Sobre o que se trabalha?

  • A representação intermédia geralmente é uma estrutura de dados (p.e. Dicionário);
  • O texto é dado por uma lista de chars;
  • O que pretendemos obter a partir do texto é uma lista de tokens;
  • Um token possui semântica associada.

Expressões Regulares (RegEx)

Podemos utilizar o site regex101 para treinarmos as expressões e perceber o seu conteúdo.

Construtores

  • a - caractere a;
  • ab - caractere a seguido pelo caractere b;
  • () - agrupamento de caracteres;

Vamos então representar um ano em regex:

('0'|...|'9')('0'|...|'9')('0'|...|'9')('0'|...|'9')
  • a? - caractere a é opcional, ou seja, pode ou não surgir;
  • a* - caractere a surge 0 ou mais vezes repetidamente;
  • a+ - caractere a surge 1 ou mais vezes repetidamente;

Vamos então representar um inteiro, independentemente da sua dimensão, em regex:

Int = ('-'|'+')?('0'|...'9')+
  • \ - retira o significado especial de um caractere passando a ver-se esse elemento como um caractere normal.
Como podemos representar uma string binária?

(0|1)*
E uma string binária que não contém a string '011' em nenhuma parte da string?

1*(0|0+1)*
  • [ ] - classe de caracteres, pode, por exemplo representar um intervalo contendo, no seu interior, 'x-y' (a ordem é dada pela tabela ASCII)

Assim sendo, podemos aprimorar o regex que representa um inteiro:

('-'|'+')?[0-9]*
  • {min, max} - no mínimo, repete 'min' vezes, no máximo, repete 'max' vezes. Se tiver apenas um elemento, repete esse número de vezes. Se não tiver valor à esquerda é 0 e, se não tiver à direita tende para infinito.

Vamos representar uma data e um número real em regex:

data = [0-9]{4}-(0[1-9]|1[0-2])-[0-9]{2}
real = (\+|\-)?[0-9]*(\.[0-9]+)?
